This document lists all asset files in the repository with absolute URLs and includes usage guidelines for each folder.

## Metadata

### Purpose
Contains comprehensive talent and creator data for development, testing, and demonstration purposes.

**METADATA_README.md** provides detailed profiles for 45 fake content creators including biographical information, social media handles, follower counts, and platform links. This serves as the primary reference for creator identities and basic metrics.

https://proto.dev.foam.io/assets/METADATA_README.md is the direct url for metadata

**fake_talent_dataset_100.csv** contains extensive analytical data with demographic breakdowns, engagement metrics, platform-specific statistics, and performance analytics. The first 45 rows correspond to the creators in METADATA_README.md, while remaining rows contain additional synthetic data for testing scalability and bulk operations.

https://proto.dev.foam.io/assets/fake_talent_dataset_100.csv is the direct url for the csv version of metadata

### Usage Rules
Use METADATA_README.md for creator profile displays, bio information, and social platform integration. Use the CSV dataset for analytics dashboards, demographic analysis, and performance testing with large datasets.

## fonts

### Purpose
Provides consistent typography across the application using Google Fonts.

### Usage Rules
Load fonts from Google Fonts with proper preconnect for optimal performance. Use variable font weights for flexibility.

### Special Notes
Uses Hanken Grotesk variable font from Google Fonts with weights ranging from 100-900.

> **Hanken Grotesk** - Primary brand typeface for headings, body text, and UI elements.  
> **Variable font** from Google Fonts for optimal performance and flexibility.  
> **Font weights available:** Variable (100-900) - any weight value between 100 and 900.  
> **Styles available:** Normal and Italic for all weights.  
> **Preconnect** to Google Fonts for faster loading performance.

**Implementation Methods:**

**Method 1: Link Tags (Recommended)**
```html
<!-- Embed in <head> -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Hanken+Grotesk:ital,wght@0,100..900;1,100..900&display=swap" rel="stylesheet">
```

**Method 2: CSS Import**
```css
/* Embed in <head> */
@import url('https://fonts.googleapis.com/css2?family=Hanken+Grotesk:ital,wght@0,100..900;1,100..900&display=swap');
```

**Font Weight Guidelines:**
- **Light (300):** Subtle text, captions, secondary information
- **Regular (400):** Body text, paragraphs, general content
- **Medium (500):** Emphasis, subheadings, important text
- **Semi Bold (600):** Headings, titles, prominent text
- **Bold (700):** Main headings, hero text, strong emphasis
- **Italic variants:** Available for all weights - use for emphasis, quotes, or stylistic variation

**CSS Implementation:**
```css
/* Variable font approach */
.hanken-grotesk {
  font-family: "Hanken Grotesk", sans-serif;
  font-optical-sizing: auto;
  font-weight: 400; /* Use any value from 100-900 */
  font-style: normal;
}

/* Italic variant */
.hanken-grotesk-italic {
  font-family: "Hanken Grotesk", sans-serif;
  font-optical-sizing: auto;
  font-weight: 400; /* Use any value from 100-900 */
  font-style: italic;
}

/* Example weight classes */
.hanken-grotesk-light {
  font-family: "Hanken Grotesk", sans-serif;
  font-optical-sizing: auto;
  font-weight: 300;
  font-style: normal;
}

.hanken-grotesk-bold {
  font-family: "Hanken Grotesk", sans-serif;
  font-optical-sizing: auto;
  font-weight: 700;
  font-style: normal;
}
```
